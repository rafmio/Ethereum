Адрес в Эфириуме - 160-битный идентификатор, используемый для идентификации
любой учетной записи

Два вида уч.зап.:
 - внешние уч.зап., контролируются с пом.закрыт.ключей. Не имеют кода, связанного
   с ним
 - контрактрые уч.записи, контролируются спец.кодом, указанным в усл.контракта и
   имеющие связанный с ним код

Состояния учетных записей:
- nonce
- balance
- storageRoot
- codeHash

nonce
Если настоящая уч.запись соотв-ет внешней уч.записи, то полученное число предс.
собой кол.транзакций, кот.были отправлены с адреса учетной записи. Если уч.запись
явл. контрактной - то nonce - это количество контрактов, созданных в данной
уч.записи.

balance
общее количество wei, приобретенных данной уч.записью.
Каждый эфир, кот. явл.обенной единицей Эфириумя, содержит 10^6 wei - дробных частей
эфира

storageRoot
хэш корневого узла префиксного дерева Меркла. Дерево Меркла кодирует хэш
содержимого данной уч.записи, при этом по умолчанию оно является пустым

codeHash
хэш EVM-кода (Ethereum Virtual Machine) учетной записи. Для контрактных уч.
записей данное поле явл.кодом, кот.хэшируется и хранится в виде codeHash
--------
Состояние Эфериума -  сопоставление адресам учетной записи состояний счета.
Это сопоставление хранится в структуре данных - префиксного дерева Меркла

Дерево Меркла (Merkle Trie) - тип двоичного файла, состоящего из наборов
узлов, которые включают:
  - определенное количество листовых узлов, кот.распологаются в нижней части
    дерева, содержащего базовые данные
  - набор промежуточных узлов, при этом каждый узел предст.собой хэш двух его
    дочерних узлов
  - один корневой узел, также образованный из хэша двух дочерних узлов, который
    представляет вершину дерева

-------
За любое вычисление, осуществлющееся в результате проведения операций с транзакциями
внутри сети Эфириума, берется определенная плата.
Номинал данной оплаты носит название "gas" ("горючее").
Горючее - единица измерения, используемая для определения размера оплаты по
конктретному вычислению.
Цена на горючее (gas) - это количество эфира, которое вы способны потратить на
каждую единицу горючего. Имеряется в gwei.
Wei - самая маленькая единица эфира
1018 Wei = 1 эфир
1 gwei = 1 000 000 000 Wei
------

Любая операция, кот.выполняется сетью, также одновременно выполняетс каждым
полным узлом.
-------

Существуют два типа транзакций:
  - отправка сообщений
  - создание контракта
-------

Все транзакции содержат следующие элементы:
  - nonce - количество транзакций, которые были отправлены отправителем
  - gasPrice - количество Wei, кот. отправитель готов отдать за единицу горючего
    за совершение сделки
  - gasLimit - максимальное количество горючего, кот.отправитель готов заплатить
    за проведение данной транзакции
  - to  - адрес получателя
  - value - кол. Wei, кот будут переданы от отправителя получателю
  - v, r, s - обозначения для создания подписи, кот.индентифицирует отправителя
  - init - только для транзакций,связ. с созд.контрактов
  - data - входные данные (параметны) для вызова сообщения (data явл. необяз.
    элементом, предназначен только для вызова соощений)

------
Все транзакции так или иначе сгруппированы в блоки. Блокчейн содержит несколько
таких блоков, соединенных между собой
Блоки состоят из
  - заголовка блока
  - информации о серии транзакций, включенных в данный блок
  - серии других заголовков блоков для текущих номеров
